# Phase 1 Context: Core Devotee & Seva

## Domain Boundaries
Devotee registration, seva booking, and tracking fulfilment.

## Locked Decisions
These choices are locked and should not be changed by downstream agents.

**Devotee Lookup UX**:
Universal Auto-Complete Bar. The UI should feature a prominent search box where staff can type a name, phone, or gotra, and a dropdown of matches appears for selection.

**Carried over from Project Initialization**:
- UI must support English and Kannada toggling.
- Voice dictation is prioritized for rapid data entry by non-tech-savvy staff.

## Planner Discretion
The following areas were explicitly skipped during discussion and are left to the planner's technical and UX discretion:
- **Booking Workflow**: Planner to decide between a multi-step wizard or a single unified form based on best UX.
- **Fulfilment Tracking**: Planner to decide the optimal way for staff to mark sevas as completed (e.g., daily batch checkboxes vs individual toggles).
- **Voice Integration**: Planner to determine exact placement of the microphone UI element (e.g., global vs per-field).
