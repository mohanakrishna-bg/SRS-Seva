| Token | Value | Usage |

| sm | 8px | Button/Input internal padding, inline card gaps |
| md | 16px | Card content spacing, general grid gaps |
| lg | 24px | Page padding, modal margins, large desktop sections |

## Visual Focal Points & Hierarchy

### Primary Screen: Accounting Dashboard & Ledger Drill-down
- **Visual Focal Point (Primary Anchor)**: The large balanced aggregate "Total Cash & Bank Assets Card" at the top of the dashboard draws the eye first, followed by the "Account Heads" list.
- **Hierarchy Pattern**: Bold 20px headers for module panels, styled light-golden borders wrapping key ledgers, and status badge overlays highlighting unreconciled funds.
- **Accessibility Guarantee**: Icon-only actions (such as standard edit/delete/reconcile checkmarks) will ALWAYS include explicit aria-labels and descriptive tooltip fallback labels.

### Form Screen: Payment Voucher
- **Visual Focal Point**: The primary action button card at the bottom right housing the prominent Saffron-colored "Save Payment Voucher" CTA.
- **Hierarchy Pattern**: Top-to-bottom standard layout grid, clear 14px bold labels, and a subtle saffron glow highlighting the active narration text input field.

## Interactive Interface Designs

### 1. Payment Voucher Form (Granular Expense Capture)
- **Layout**: Two-column grid on desktop, stacking to single-column on mobile.
- **Elements**:
  - **Payee Name**: Text input with placeholder `"Enter vendor or payee name"`
  - **Date**: Input type `date`, defaulting to current local date.
  - **Amount**: Numerical input with a prefixed absolute label `"INR"` and `placeholder="0.00"`.
  - **Category**: Hierarchical select box displaying account heads prefixed with indentation dots (e.g. `Prasada`, `..Prasada Ingredients`).
  - **Source Account**: Select box toggling between `Cash Account (A001)` and `Bank Account (A002)`.
  - **Reference ID**: Text input for UTR, Cheque number, or internal cash reference.
  - **Narration**: Textarea with a microphone icon button floating on the right side for voice dictation. Active dictation is indicated by a glowing saffron shadow (`shadow-amber-500/50`).
  - **Image Link**: Text input supporting external URLs for uploaded bill images.
- **Validation**:
  - Amount must be greater than zero.
  - Category and Payee Name are strictly required.
  - Visual error outlines: red border (`border-[var(--color-expense)]`) with help text.

### 2. Bank Reconciliation Manual Board
- **Layout**: Row-based tabular layout with standard columns: Date, Description, Reference, Amount, Status Badge, Actions.
- **Statuses**:
  - **Pending Verification**: `Warning` styled outline badge: text Saffron, border Saffron, transparent background. Actions: green check icon `"Clear / Reconcile"`, red cross icon `"Mark Bounced"`.
  - **Reconciled**: `Success` styled solid badge: text white, background emerald, border emerald. Actions: none (locked icon).
  - **Bounced / Failed**: `Destructive` styled solid badge: text white, background red/burgundy, border red/burgundy. Actions: none (locked icon).
- **Enforcement Notice**: A prominent yellow header bar warns staff: *"Note: Uncleared funds are excluded from the active balance until manually marked as Reconciled."*

### 3. Print media overrides (`@media print`)
- **Action**: Native print styles applied inside the global `index.css` sheet.
- **Constraints**:
  - All navigation drawers, user menus, filters, page title action buttons, and footer sections are hidden (`display: none !important`).
  - Page backdrop colors are forced to white and text colors to charcoal-black to ensure crisp high-contrast printouts.
  - Tables utilize native CSS page-breaks: `tr { page-break-inside: avoid; }` to prevent text splitting across PDF page boundaries.
  - Margins set explicitly: `@page { size: portrait; margin: 1.5cm; }` for A4 compatibility.
