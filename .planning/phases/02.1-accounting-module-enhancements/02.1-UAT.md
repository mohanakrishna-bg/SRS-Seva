## 2. Verification Log

### UAT-2.1.1: Expense capture
- [x] Category dropdown displays hierarchical expense categories with prefix indentation dots.
- [x] Voucher forms block submissions when Payee, Category, or positive Amount are missing.
- [x] Submitting a voucher posts a balanced double-entry JournalEntry (Debit Expense Account, Credit Asset Account).
- [x] Dictating narrations via mic button correctly transcribes speech and highlights input border with glowing saffron.

### UAT-2.1.2: Bank reconciliation
- [x] Tabular bank reconciliation panel is displayed under the Reconciliation tab.
- [x] Status badges are styled in accordance with the UI-SPEC design variables.
- [x] Balance only increases or decreases when status transitions to `Reconciled` (marked as cleared).
- [x] Bounced transaction states reverse balances and display correct indicators.

### UAT-2.1.3: Day-close lock
- [x] Administrative "Close Daily Books" action is displayed on the dashboard for admin roles only.
- [x] Confirmation prompt is triggered before locking.
- [x] Database lists locked date in `ClosedDay` table.
- [x] Attempting write operations on a locked date returns standard `403 Forbidden` errors.

### UAT-2.1.4: Grace period
- [x] Counter staff can edit/cancel a transaction within 2 hours of creation.
- [x] Edits to transactions created > 2 hours ago by standard staff are rejected with `403 Forbidden`.
- [x] Administrators can edit transactions beyond the rolling 2-hour window.

### UAT-2.1.5: Print media optimization
- [x] Media print styles strip navbars, header bars, and download buttons cleanly.
- [x] Report structures center properly on portrait A4 layouts.
- [x] Background gradients are simplified to solid white with crisp charcoal-black texts.
