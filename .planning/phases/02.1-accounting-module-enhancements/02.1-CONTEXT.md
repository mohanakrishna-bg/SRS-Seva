# Phase 2.1: Accounting Module Enhancements - Context

**Gathered:** 2026-05-27
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase delivers core visual, functional, and reporting enhancements to the Financials & Accounting module of **SRS-Seva**. It includes granular expense tracking, interactive sub-ledger drill-down views, manual bank transaction reconciliation, audit trail day-closing locks, and client-side data exports.

</domain>

<decisions>
## Implementation Decisions

### 1. 🔍 Interactive Ledger Drill-Down & Search (REP-02)
- **D-01 (Row Click Details):** Clicking any transaction line in the Ledger or Journal table opens a comprehensive details popover modal showing Devotee name, Gotra/Nakshatra, Seva details, payment logs, and original receipt references.
- **D-02 (Account Drill-Down):** Clicking on any Account Head (e.g., "Cash in Hand", "UPI Receipts", or specific Expense heads) in the Ledger dynamically loads a dedicated chronological double-entry statement card showing all Debit and Credit lines that compose the total.
- **D-03 (Advanced Filters):** Build a filter toolbar allowing staff to search the ledger and journal entries by Devotee Name, Gotra, Receipt ID, amount ranges, and Payment Mode.

### 2. 🏦 Granular Expense Capture (FIN-03)
- **D-04 (Expense Voucher Form):** Create a dedicated **Payment Voucher Form** in the UI to log operational expenditures.
- **D-05 (Fields Captured):** The voucher form must capture:
  - Payee / Vendor Name
  - Date of Expense
  - Expense Category (Account Head selection from hierarchical dropdown)
  - Amount (INR)
  - Source Payment Account (Cash drawer or Bank account remittance)
  - Reference number (e.g., Cheque No, Bank UTR, or cash receipt ID)
  - Narration / Description
  - External Receipt Image Link (allowing users to paste image URLs)
- **D-06 (Double-entry auto-post):** Saving an expense automatically posts a balanced Journal Entry (Debits the selected Expense Head, Credits the selected Cash/Bank asset head) and adjusts bank/cash balances accordingly.

### 3. 🏦 Bank Reconciliation & Verification (FIN-02)
- **D-07 (Manual Reconciliation Board):** Introduce a reconciliation tab/view inside the "Bank & Cash" section. Staff can view all bank transactions (cheque, transfer, UPI) and toggle their reconciliation status between:
  - `Pending Verification`
  - `Reconciled` (marked as cleared)
  - `Bounced / Failed`
- **D-08 (Balance Enforcement):** Only `Reconciled` bank transactions are fully added to the Bank Account's `CurrentBalance`. `Pending Verification` items are shown as "Uncleared Funds" but do not alter the main active balance.

### 4. 📄 Accounting Reports & Audit Exporting (REP-02)
- **D-09 (Client-side Data Exports):** Add "Export to Excel/CSV" buttons to the Journal ledger, Bank transactions table, and daily collection breakdown grids.
- **D-10 (Print-Optimized PDFs):** Style the Income & Expenditure Statement and Balance Sheet snapshot with custom print CSS (`@media print`) so administrators can natively save highly-polished, formal PDF reports.
- **D-11 (Receipt Bundling):** Staff can consolidated all receipts booked on a specific date into a single merged PDF package for audit sign-offs.

### 🔒 5. Daily Day-Closing & Audit Locks (FIN-03)
- **D-12 (Administrative Close Day):** Implement an administrative action button "Close Daily Books". Clicking this locks all entries (Receipts, Journal Lines, Expenses) for that date. Locked entries are completely read-only and cannot be edited, cancelled, or deleted by any user role.
- **D-13 (Staff Edit Grace Period):** General counter staff have a rolling **2-hour grace period** from the moment of creating a transaction to make minor corrections or cancel it, after which it locks automatically unless explicitly unlocked or modified by an Administrator.

### the agent's Discretion
- Hierarchical Account Head dropdown layout and grouping of sub-categories.
- Detail Modal layout structure (standard modal layout with responsive grid details).
- Visual status badges for reconciliation (`text-amber-500` for pending, `text-emerald-500` for reconciled, `text-red-500` for failed).

</decisions>
