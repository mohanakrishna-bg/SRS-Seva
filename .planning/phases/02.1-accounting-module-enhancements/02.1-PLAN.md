# Phase 02.1: Accounting Module Enhancements

## 1. Goal

Implement comprehensive visual, functional, and reporting upgrades to the Financials & Accounting module of **SRS-Seva**. This includes double-entry auto-posted operational expense vouchers, manual bank reconciliation with active balance enforcement, daily day-closing audit locks, counter staff rolling 2-hour cancelation grace periods, print-optimized media outputs, and interactive sub-ledger drill-down navigation.

---

## 2. Requirements

*   **FIN-02 / FIN-03 (Granular Expense Vouchers)**: Log payee, category (hierarchical select), source (Cash/Bank), references, voice-dictated narration, and bills. Auto-post balanced debit expense and credit cash/bank journal lines.
*   **FIN-02 (Bank Reconciliation)**: Staff reconciliation board supporting transaction states: `Pending Verification`, `Reconciled` (cleared), `Bounced / Failed`. Uncleared transactions do not alter active balance.
*   **FIN-03 (Audit Day Close & Grace Period)**: Day-close button locks entries for that date permanently. Standard counter staff have a rolling 2-hour grace period for editing or canceling entries.
*   **REP-02 (Ledger Drill-down & Exports)**: Row click shows nested breakdowns. Account head click loads sub-ledgers. Export ledger data to CSV.
*   **REP-02 (Print Optimization)**: CSS print styles for clean, black-on-white native PDF exports of statements. Merge receipts by date.

---

<read_first>
- backend/app/api/accounting.py
- backend/app/models/accounting.py

<action>
1. In `backend/app/api/accounting.py`, implement helper function `check_date_locked(entry_date: str, db: Session)`:
   ```python
   def check_date_locked(entry_date: str, db: Session):
       is_locked = db.query(accounting.ClosedDay).filter(accounting.ClosedDay.Date == entry_date).first()
       if is_locked:
           raise HTTPException(status_code=403, detail="Transaction Locked: This date's books have been signed off. Modifications are disabled.")
   ```
2. Implement admin endpoint to lock books for a given day:
   ```python
   @router.post("/close-day", response_model=schemas.ClosedDay)
   def close_day(closed: schemas.ClosedDayCreate, db: Session = Depends(database.get_db)):
       db_closed = accounting.ClosedDay(Date=closed.Date)
       db.add(db_closed)
       db.commit()
       db.refresh(db_closed)
       return db_closed
   ```
3. Implement counter staff rolling 2-hour edit/delete grace period check:
   - For all `PUT` or `DELETE` endpoints in `backend/app/api/accounting.py` and `backend/app/main.py` (Seva bookings):
     - Check the record's `CreatedAt` date. If the time difference between current UTC datetime and the record's `CreatedAt` is greater than 2 hours (`datetime.datetime.utcnow() - record.CreatedAt > datetime.timedelta(hours=2)`):
       - If user is standard staff (not admin), raise `403 Forbidden` ("The rolling 2-hour edit grace period has expired. Please contact an Administrator to unlock this transaction").
4. Incorporate `check_date_locked` at the top of ALL write operations (`POST`, `PUT`, `DELETE` routes on `journal`, `bank-transactions`, `accounts`, and `/api/seva` booking registrations).
</action>
<acceptance_criteria>
- `backend/app/api/accounting.py` contains `def check_date_locked` and `/close-day` route
- Write routes raise `403 Forbidden` when attempting transactions on a date present in `ClosedDay` table
- Non-admin modifications to transactions created > 2 hours ago raise `403 Forbidden`
