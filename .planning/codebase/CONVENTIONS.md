# Coding Conventions

## Frontend
- **Language**: TypeScript (`.ts`, `.tsx`) for strong typing.
- **Components**: Functional components with hooks.
- **Styling**: Utility-first CSS using Tailwind CSS. Use of standard Glassmorphism UI patterns.
- **Formatting**: ESLint is used for linting (`npm run lint`).
- **Naming**: `camelCase` for variables and functions, `PascalCase` for React components.

## Backend
- **Language**: Python 3.
- **API Design**: RESTful API conventions with FastAPI.
- **Typing**: Python type hints extensively used with Pydantic for validation.
- **Naming**: `snake_case` for variables, functions, and module names. `PascalCase` for classes (Models, Schemas).
